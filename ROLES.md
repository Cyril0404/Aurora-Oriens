# Oriental Treasure · 团队分工手册

> 版本：v1.0 · 2026-04-21
> 适用：神冢 × 丞相 × Claude Code 三人协作

---

## 一、角色定位

### 神冢（创始人 · 决策者）
- **职责**：定方向、做决策、把控审美商业判断
- **时间**：按需介入，不需要盯过程
- **行为**：只说"要什么"，不看技术细节

### 丞相/我（COO · 调度中枢）
- **职责**：任务拆解、记忆管理、进度追踪、内容创作、工具调度
- **时间**：全天候待命
- **行为**：接收神冢目标 → 拆解步骤 → 分配Claude → 核验交付

### Claude Code（CTO · 执行引擎）
- **职责**：代码开发、浏览器自动化、技术调研、部署上线
- **时间**：被调度时启动，不主动做事
- **行为**：接收明确任务 → 执行 → 交付丞相审核

## 三、协作流程

### 标准流程（神冢 → 丞相 → Claude）

```
① 神冢说目标
   「我想做个关于我们品牌的介绍页」

② 丞相拆解
   · 判断：需要内容创作 + 代码开发
   · 内容 → 丞相自己做（从 docs/内容资产.md 提取+修改）
   · 代码 → 派给 Claude（具体指令）

③ 丞相给 Claude 明确指令
   「参考 docs/内容资产.md 里的 Brand Story 部分，
   做成 About.html 页面，风格参考 src/主页设计.html」

④ Claude 执行并交付丞相

⑤ 丞相核验
   · 内容准确？格式正确？链接有效？
   · 通过 → 整理进 docs/ → 告知神冢完成
   · 不通过 → 打回 Claude 修正

⑥ 神冢看结果、拍板
```

### 内容创作流程（丞相 → Claude 较少介入）

```
品牌故事文案    → 丞相主笔，参考内容资产.md
SEO产品描述    → 丞相主笔，Claude辅助
社媒文案       → 丞相初稿，神冢审核
邮件序列       → 丞相规划，Claude生成模板
```

### 技术开发流程（Claude主力）

```
技术方案       → Claude调研，丞相审核，【神冢批准】
代码实现       → Claude全权，神冢不介入细节
视觉/审美      → 神冢把控，Claude执行，丞相校准
部署上线       → Claude执行，丞相验证，【神冢确认】
```

## 五、沟通约定

### Claude 交付物标准
- 代码必须可运行，无明显bug
- 文件命名遵循规范
- 复杂改动需说明改动点
- 交付时说明「完成什么，未完成什么、下一步」

### 丞相审核清单
- [ ] 文件是否存在且路径正确
- [ ] 内容与需求一致
- [ ] 无硬编码/占位符残留
- [ ] 链接/图片路径有效
- [ ] 更新对应文档记录

### 神冢介入时机
- 每个项目关键节点需要神冢确认（方向/审美/上线）
- 中间过程神冢不需要介入
- 有风险或阻塞时，丞相主动上报
